Complete Detailed Project Specification

1. User Authentication & Profile

Features:

  • **User Registration:**
    • Fields: Full Name, Email, Password (with confirmation)
    • Validations: Email format, password strength
    • Passwords hashed (bcrypt or similar)
  • **User Login:** Email + Password verification, Use session (express-session) or JWT tokens
  • **Logout**
  • **Profile Management:** View & update profile (Name, Email), Change password with old password verification
  • **Security:** Prevent duplicate emails, Password hashing and salting, Input sanitization to prevent injection attacks

UI:

  • Registration page with clear form and validation feedback
  • Login page with “forgot password” link (optional)
  • Profile page showing user info and editable fields
  • Secure routes: dashboard and listing management only accessible after login

2. Property Listings

Features:

  • **Create Listing:**
    • Title (string, required)
    • City (string, required)
    • Rent (number, required)
    • Property Type (dropdown: Apartment, House, PG, Shared Room, Studio)
    • Photos: multiple image URLs input (comma separated or multiple fields)
    • Description (optional)
  • **Edit Listing:** Only owner can edit
  • **Delete Listing:** Only owner can delete
  • **View Listings:** List all listings with summary info (title, city, rent, type), Click listing to view full details
  • **Upload/Store photos:** Initially URLs for simplicity, later can add actual image upload using multer and local/cloud storage

UI:

  • Form for adding/editing listings with validations
  • Listings displayed in card/grid layout with photos thumbnails
  • Property details page showing larger images gallery and full info

3. Search & Filters

Features:

  • Search bar filtering listings by city or keywords in title
  • **Filters by:** Rent range (min/max), Property type
  • **Sort listings by:** Rent ascending/descending, Date added (newest first)
  • Pagination or infinite scroll for large results (optional)

UI:

  • Search bar at top of home or listings page
  • Sidebar or dropdown filters
  • Results dynamically update as filters change (using jQuery AJAX)

4. User Dashboard

Features:

  • After login, user can access dashboard
  • **Dashboard shows:**
    • All user’s listings with quick summary (title, city, rent)
    • Buttons to edit or delete each listing
    • Button to add new listing
    • Show messages or notifications (optional)

UI:

  • Clean, organized dashboard with listing cards or table
  • Confirmation dialogs on delete
  • Status messages for add/edit/delete actions

5. Home / Landing Page

Features:

  • Welcome message with catchy tagline
  • Search bar for quick property search
  • Preview of latest or featured listings (e.g., 5 most recent)
  • Brief about Apna Adda platform & its vision

UI:

  • Warm colors, Indian-inspired typography
  • Simple, inviting layout
  • Responsive design for mobile and desktop

6. Listing Details Page

Features:

  • **Show all details of a property:** Title, City, Rent, Property Type, Description, Photo gallery/carousel
  • **Optionally:** Contact details of the lister, Button to send inquiry (email or form)

UI:

  • Large header with title
  • Photo gallery with thumbnails or slideshow (using jQuery plugin or custom)
  • Clean layout with labeled info fields

7. Notifications & Messaging (Optional for MVP)

Features:

  • Users receive notifications for inquiries or interest shown in listings
  • Simple contact form on listing detail to send message to owner
  • Inbox view in dashboard to manage messages

9. MongoDB Schemas

User Schema

const UserSchema = new mongoose.Schema({
  name: { type: String, required: true },
  email: { type: String, required: true, unique: true },
  passwordHash: { type: String, required: true },
  createdAt: { type: Date, default: Date.now }
});

Listing Schema

const ListingSchema = new mongoose.Schema({
  ownerId: { type: mongoose.Schema.Types.ObjectId, ref: 'User', required: true },
  title: { type: String, required: true },
  city: { type: String, required: true },
  rent: { type: Number, required: true },
  propertyType: { type: String, required: true },
  photos: [String], // Array of image URLs
  description: { type: String },
  createdAt: { type: Date, default: Date.now }
});

11. UI/UX Design Details

  • **Colors:** Earthy warm colors — ochre, burnt orange, mustard yellow, cream backgrounds.
  • **Fonts:** Combination of modern sans serif for body and elegant serif (like ‘Georgia’ or ‘Merriweather’) for headings.
  • **Buttons:** Rounded corners, hover effects with deeper shades of orange.
  • **Forms:** Clean inputs with subtle shadows, error highlighting in red, success messages in green.
  • **Responsive:** Layout adjusts gracefully for mobile screens; hamburger menu on small devices.
  • **Animations:** Subtle fade-ins for listings, hover scale on cards.

12. Security Considerations

  • Hash passwords securely (bcrypt)
  • Sanitize inputs to avoid XSS/Injection attacks
  • Protect routes with middleware for authentication
  • Limit rate of requests to avoid brute force attacks on login
  • HTTPS for secure data transfer (optional on deployment)
